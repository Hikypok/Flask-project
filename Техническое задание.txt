ТЕХНИЧЕСКОЕ ЗАДАНИЕ НА РАЗРАБОТКУ ВЕБ-ПЛАТФОРМЫ "ПРИЯТНЫЙ ДОСУГ"

1. ОБЩИЕ СВЕДЕНИЯ И ЦЕЛИ ПРОЕКТА
Настоящее техническое задание определяет требования к разработке веб-платформы "Приятный досуг", предназначенной для агрегации пользовательских отзывов о культурных объектах и местах отдыха. Основной целью является создание удобного сервиса, позволяющего пользователям делиться мнениями о книгах, фильмах и местах для посещения, а также искать рекомендации на основе рейтингов и текстовых описаний. Система должна обеспечивать интуитивно понятный интерфейс, адаптивный дизайн и надежное хранение данных. Технологический стек включает язык Python, фреймворк Flask, ORM SQLAlchemy для работы с базой данных, шаблонизатор Jinja2 для генерации HTML-страниц, CSS-фреймворк Bootstrap для верстки и библиотеку Pillow для обработки изображений. Для отображения географических данных в разделе мест используется API Яндекс Карт.

2. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ К МОДУЛЯМ СИСТЕМЫ

2.1. Модуль аутентификации и управления пользователями
Система должна поддерживать регистрацию новых пользователей и авторизацию зарегистрированных. Процесс регистрации требует ввода уникального идентификатора (никнейма), адреса электронной почты и пароля. Вход в систему осуществляется по учетным данным. После входа пользователь получает доступ к личному кабинету. Личный кабинет включает возможность просмотра и редактирования профиля: изменения никнейма и загрузки аватара. При загрузке аватара система должна автоматически обрабатывать изображение (масштабирование, обрезка) с помощью библиотеки Pillow для обеспечения единообразного отображения в интерфейсе. Профиль пользователя должен содержать список всех опубликованных им отзывов.

2.2. Модуль управления контентом (Отзывы)
Основной функциональной единицей системы является пост-отзыв. Каждый отзыв привязывается к одному из трех разделов: "Что бы мне почитать" (Книги), "Что бы мне посмотреть" (Фильмы) или "Куда бы мне пойти" (Места). При создании отзыва пользователь обязан указать название объекта, текст отзыва и выставить рейтинг в диапазоне от 1 до 10. Создание, редактирование и удаление отзывов доступны только авторизованным пользователям и только для собственных записей. Структура данных отзыва должна быть спроектирована с учетом возможности будущего расширения атрибутов для разных категорий контента, без жесткой фиксации схемы хранения на данном этапе, но с обязательным наличием базовых полей: заголовок, текст, рейтинг, категория, автор, дата создания.

2.3. Специфика раздела "Куда бы мне пойти" и интеграция карт
Для отзывов в категории "Места" требуется дополнительная функциональность, связанная с геолокацией. При создании такого отзыва пользователь должен иметь возможность указать местоположение объекта через интерфейс Яндекс Карт (выбор точки на карте или ввод адреса). Координаты или адрес места должны сохраняться в базе данных. При просмотре отзыва категории "Место" на странице должна отображаться интерактивная карта с меткой, указывающей на локацию объекта. 

2.4. Модуль навигации, поиска и сортировки
Главная страница и страницы разделов должны отображать списки отзывов в виде адаптированных карточек. Пользователям должна быть предоставлена возможность сортировки списков отзывов внутри каждого раздела. Доступные варианты сортировки: по дате публикации (новые/старые), по рейтингу (высокий/низкий) и по популярности. Алгоритм расчета популярности может быть реализован на основе даты обновления или других метрик, определяемых разработчиком. Также должен функционировать поиск по сайту, осуществляющий выборку отзывов по совпадению ключевых слов в заголовках и тексте отзывов. Результаты поиска должны выводиться в общем формате списка отзывов.

3. ТРЕБОВАНИЯ К ИНТЕРФЕЙСУ И ПОЛЬЗОВАТЕЛЬСКОМУ ОПЫТУ (UI/UX)
Интерфейс приложения должен быть реализован с использованием CSS-фреймворка Bootstrap, что гарантирует адаптивность под различные разрешения экранов (мобильные устройства, планшеты, десктопы). Навигационное меню должно обеспечивать быстрый доступ к основным разделам сайта, функции поиска и личному кабинету. Визуальное оформление должно быть чистым и минималистичным, акцентируя внимание на контенте. Элементы управления (кнопки, формы, выпадающие списки сортировки) должны соответствовать стандартам Bootstrap и быть интуитивно понятными. Отображение рейтинга должно быть визуально выделяющимся элементом карточки отзыва.

4. ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ И АРХИТЕКТУРА
Серверная часть приложения строится на базе Flask. Маршрутизация и логика обработки запросов должны быть организованы модульно (например, с использованием Blueprints) для удобства поддержки и масштабирования. Работа с базой данных осуществляется исключительно через ORM SQLAlchemy, что позволяет абстрагироваться от конкретной СУБД и легко менять структуру таблиц при необходимости. Шаблоны страниц создаются на Jinja2 с использованием механизма наследования для соблюдения единого стиля layout. Обработка файлов аватаров производится на сервере с использованием Pillow, файлы сохраняются в статической директории проекта, а в базе данных хранятся только ссылки на них. Интеграция с Яндекс Картами выполняется на стороне клиента (JavaScript) с передачей необходимых данных на сервер для сохранения.

5. ТРЕБОВАНИЯ К БЕЗОПАСНОСТИ
Система должна обеспечивать безопасное хранение паролей пользователей с использованием современных алгоритмов хэширования. Все формы отправки данных должны быть защищены от CSRF-атак. Входные данные от пользователей должны проходить валидацию и санитизацию для предотвращения XSS-атак и инъекций. Доступ к функциям редактирования и удаления контента должен быть строго ограничен правами автора записи.

6. КРИТЕРИИ ПРИЕМКИ
Работоспособность системы подтверждается успешным выполнением следующих сценариев: регистрация и вход пользователя; загрузка и отображение аватара; создание отзывов во всех трех категориях; корректное сохранение и отображение карты для отзывов о местах; работа сортировки и поиска; адаптивное отображение интерфейса на мобильных устройствах. Код должен быть структурирован, документирован и готов к дальнейшему расширению функционала.
